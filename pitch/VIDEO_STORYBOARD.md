# PRONTO — Video Storyboard (60-90 seconds)

## Format & Tone
- **Style:** Clean, fast, cinematic product shots with bold text overlays
- **Music:** Electronic, building momentum — think Apple keynote energy
- **Voice:** Single narrator, confident and direct, no hype words
- **Resolution:** 1920x1080, 30fps minimum
- **Target length:** 75 seconds (sweet spot)

---

## SCENE 1: THE PAIN (0:00 — 0:15)

**Visual:** Fast montage — developer staring at a blank ChatGPT text box, Slack messages flying ("this prompt doesn't work"), a terminal full of failed API calls, someone copy-pasting prompts between tabs.

**Text overlay (bold, sequential):**
```
30 MILLION DEVELOPERS BUILD WITH AI.
EVERY ONE OF THEM WRITES PROMPTS.
NONE OF THEM HAVE THE RIGHT TOOL.
```

**Voiceover:**
> "Every developer building with AI hits the same wall. Prompts are treated like throwaway strings — no testing, no versioning, no way to know if they work until they're live and breaking things."

**Sound:** Typing sounds, notification pings, building tension.

## Production Notes

### Shot List
| # | Shot | Type | Duration |
|---|------|------|----------|
| 1 | Developer frustration montage | Screen recordings + stock | 15s |
| 2 | Black screen — "Until Now" | Motion graphics | 5s |
| 3 | Pronto logo + editor reveal | Screen capture + motion | 10s |
| 4 | Write — editor close-up | Screen capture | 10s |
| 5 | Test — multi-model comparison | Screen capture | 12s |
| 6 | Optimize — suggestion + re-run | Screen capture | 10s |
| 7 | Ship — deploy + API | Screen capture | 8s |
| 8 | Architecture diagram | Motion graphics | 5s |
| 9 | Dashboard pullback + logo | Screen capture + motion | 10s |

### Assets Needed
- [ ] Pronto product screen recordings (editor, test, optimize, ship flows)
- [ ] Pronto logo (SVG, transparent background)
- [ ] Architecture diagram (animated or static)
- [ ] Background music (licensed, upbeat electronic, 75-80 BPM building to 120 BPM)
- [ ] Voiceover recording (professional or team member with clear delivery)
- [ ] "Developer frustration" b-roll (screen recordings of messy prompt workflows)

### Editing Guidelines
- Cut on the beat — every transition should land with the music
- Text overlays: bold sans-serif (Inter or SF Pro), white on dark, left-aligned
- No more than 8 words per text overlay
- Every product shot should show real data, not placeholder text
- End card holds for 5 full seconds with logo and URL
